# Evaluation Framework

## Objective

Evaluate programming languages for the AI era: a period where software is increasingly written, modified, reviewed, tested, and maintained by humans working with AI systems and autonomous agents.

## Primary question

Which programming languages become more valuable when AI systems participate deeply in software development?

## Evaluation dimensions

| Dimension | Weight | Key question |
|---|---:|---|
| Human cognition | 20% | Can humans understand and govern the code? |
| Machine cognition | 25% | Can tools and AI systems reason about the code? |
| AI-agent operability | 25% | Can agents safely modify and verify the code? |
| Runtime and ecosystem | 20% | Can the language support production systems? |
| Strategic viability | 10% | Will the language remain relevant and supported? |

## Cross-cutting lenses

1. Verification advantage
2. Agentic development advantage
3. Legacy gravity
4. Safety pressure
5. Abstraction compression
