# C++

## AI-era hypothesis

C++ sharpens the *legacy gravity* and *safety pressure* lenses: it dominates performance-critical and ML-internals work and has a vast existing codebase, but AI-generated changes carry an unusually high blast radius due to memory unsafety and undefined behavior.

## Strengths

- Industrial dominance in systems, games, embedded, HFT, and ML-internals
- Highest practical performance ceiling among mainstream languages
- Mature compiler toolchains (Clang, GCC, MSVC) and analyzers
- Deep training-data footprint
- Vast existing codebase and library coverage

## Weaknesses

- Memory unsafety and undefined behavior raise the cost of incorrect AI changes
- Build-system fragmentation (CMake, Bazel, Make, Meson, ad hoc) complicates automation
- Long compile times slow AI feedback loops
- Header-based modularity creates weak boundaries; modules adoption uneven
- Template metaprogramming and macros obscure structure from analyzers

## Open questions

- Can AI agents safely refactor large legacy C++ codebases?
- Do modern C++ standards (C++20/23, modules, concepts) materially improve machine cognition?
- Does Rust displace C++ in greenfield AI-era systems work?
