# Rust

## AI-era hypothesis

Rust is strategically advantaged where AI-generated or AI-maintained software requires memory safety, static analyzability, and strong compiler feedback.

## Strengths

- Strong static typing
- Ownership and borrowing model
- Compiler-enforced memory safety
- Strong tooling conventions
- Strong fit for systems, infrastructure, embedded, and security-sensitive software

## Weaknesses

- Higher learning curve
- Complex generated-code repair loops
- Slower prototyping than Python or TypeScript

## Open questions

- Does AI reduce the human cost of Rust’s learning curve?
- Will agents handle borrow-checker errors reliably?
- Does generated-code volume increase Rust’s relative value?
